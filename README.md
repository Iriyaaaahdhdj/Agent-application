# 课程资料智能问答与学习总结助手

## 一、项目简介

本项目是《AI智能体设计开发及应用》课程大作业，项目名称为“课程资料智能问答与学习总结助手”。

项目仓库地址：https://github.com/Iriyaaaahdhdj/Agent-application

本项目面向大学生日常课程学习场景，设计并实现一个能够读取课程资料、理解学生问题、生成学习摘要、整理知识点、制定复习建议，并可结合飞书进行学习内容归档或通知推送的 AI 智能体应用。

该智能体旨在帮助学生提高课程资料整理效率，降低复习时的信息查找成本，让学习过程更加系统化、自动化和可追踪。

## 二、服务对象

本项目主要服务对象为：

- 大学生
- 课程学习者
- 需要整理课堂笔记、课程资料和复习计划的学生
- 需要快速回顾知识点、生成学习总结的用户

## 三、真实场景痛点

在日常学习中，学生经常会遇到以下问题：

1. 课程资料数量较多，包括课堂笔记、PPT 内容、教材片段、实验说明等，人工整理耗时较长。
2. 复习时难以快速找到重点内容和核心知识点。
3. 遇到问题时，需要反复翻阅资料，查找效率较低。
4. 学习总结和复习计划通常依赖人工完成，容易遗漏重点。
5. 学习资料缺少统一归档和提醒机制，不方便持续跟踪学习进度。

因此，本项目希望通过 AI 智能体帮助学生完成“资料理解、重点提取、问题回答、复习建议生成、学习报告输出”等任务。

## 四、用户输入

用户可以向系统输入以下内容：

- 课程笔记文本
- 教材或课件片段
- Markdown 格式的课程资料
- 学生提出的学习问题
- 需要总结的课程主题
- 可选的飞书机器人 Webhook 地址

示例输入：

```text
课程资料：人工智能智能体是能够感知环境、进行决策并执行动作的系统……
用户问题：智能体和普通聊天机器人的区别是什么？
```

## 五、智能体输出

系统根据输入内容生成以下结果：

- 课程资料摘要
- 核心知识点列表
- 用户问题回答
- 重点概念解释
- 复习建议
- 学习计划
- Markdown 格式学习报告
- 飞书通知文本或归档内容

示例输出：

```text
课程摘要：
本节内容主要介绍了 AI 智能体的基本概念、组成结构和应用场景。

核心知识点：
1. 智能体的定义
2. 感知、决策、执行三个基本环节
3. 智能体与普通聊天机器人的区别
4. 智能体在学习、办公、校园服务中的应用

问题回答：
智能体不仅能回答问题，还可以根据目标调用工具、执行任务、生成结果并进行多步骤处理。
```

## 六、核心功能模块

### 1. 资料读取模块

负责读取本地课程资料文件，例如 `data/sample_course_notes.md`，并将文本内容传递给智能体处理。

### 2. 文本清洗模块

对输入内容进行基础处理，包括去除空行、合并文本、检测资料是否为空等。

### 3. 课程摘要模块

调用大模型或模拟 AI 处理函数，对课程资料进行总结，生成简洁的学习摘要。

### 4. 知识点提取模块

从课程资料中提取重要概念、关键词、知识点和复习重点。

### 5. 智能问答模块

根据课程资料和用户问题生成回答。回答应尽量基于已有资料，避免编造无依据内容。

### 6. 复习建议模块

根据资料内容和问题类型，为学生生成个性化复习建议。

### 7. 报告生成模块

将摘要、知识点、问答结果和复习建议整理成 Markdown 格式学习报告，并保存到 `outputs/summary_report.md`。

### 8. 飞书协同模块

支持将学习总结、复习提醒或报告摘要发送到飞书群机器人，或者模拟生成飞书通知内容。

### 9. CLI 运行模块

用户可以通过命令行运行项目，完成资料读取、问题输入、报告生成和通知推送等操作。

## 七、智能体处理流程

本项目智能体的整体处理流程如下：

1. 用户准备课程资料文件，并在命令行输入学习问题。
2. 系统读取课程资料内容。
3. 系统检查输入是否为空、问题是否有效。
4. 文本清洗模块对资料进行预处理。
5. 大模型对课程资料进行摘要生成。
6. 大模型提取课程核心知识点。
7. 大模型结合课程资料回答用户问题。
8. 大模型生成复习建议和学习计划。
9. 系统将处理结果整理为 Markdown 学习报告。
10. 系统将报告保存到本地输出目录。
11. 如配置飞书 Webhook，则发送学习总结通知。
12. 如未配置 Webhook，则使用 mock 模式在命令行打印通知内容。
13. 用户根据输出报告进行复习和改进。

## 八、大模型负责的任务

大模型主要负责以下智能处理任务：

- 理解课程资料内容
- 总结课程主要内容
- 提取核心知识点
- 理解用户提出的问题
- 根据资料生成回答
- 判断问题是否超出资料范围
- 生成复习建议
- 生成学习计划
- 将零散内容整理成结构化报告

大模型在本项目中承担“理解、总结、分类、生成、问答”的核心智能任务。

## 九、工具、代码、CLI、飞书负责的任务

### 1. 工具和代码负责的任务

- 读取本地课程资料文件
- 检查输入是否合法
- 调用 AI 处理函数
- 整理模型输出结果
- 生成 Markdown 学习报告
- 保存输出文件
- 管理项目目录结构
- 提供 mock 模式，方便没有真实 API 时展示功能

### 2. CLI 负责的任务

用户可通过命令行完成项目运行，例如：

```bash
python main.py
```

或：

```bash
python main.py --file data/sample_course_notes.md --question "智能体的核心组成是什么？"
```

CLI 的作用包括：

- 安装依赖
- 启动程序
- 输入问题
- 执行脚本
- 查看输出结果
- 测试项目功能

### 3. 飞书负责的任务

飞书协同功能主要用于：

- 将学习总结发送到飞书群
- 将复习提醒推送给学生
- 将学习报告内容归档到飞书文档或飞书表格
- 方便学生后续查看、分享和协作

如果没有真实飞书 Webhook，本项目可使用 mock 模式，模拟飞书通知发送过程。

## 十、异常情况处理

系统需要考虑以下异常情况：

| 异常情况 | 处理方式 |
|---|---|
| 课程资料为空 | 提示用户补充资料，不继续生成报告 |
| 用户问题为空 | 提示用户输入明确问题 |
| 问题超出资料范围 | 提示“当前资料中未找到充分依据”，并给出可能的学习方向 |
| 文件路径错误 | 提示用户检查文件是否存在 |
| 输出目录不存在 | 自动创建 `outputs` 目录 |
| 飞书 Webhook 未配置 | 使用 mock 模式打印通知内容 |
| 大模型返回内容不完整 | 使用默认模板补全报告结构 |
| 网络或接口调用失败 | 输出错误提示，并保留本地报告生成结果 |
