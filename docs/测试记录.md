# 课程资料智能问答与学习总结助手测试记录

## 一、测试目标

本测试用于验证“课程资料智能问答与学习总结助手”在只调用豆包 API 的情况下，能否根据不同学习问题生成不同的资料摘要、核心知识点、问题回答、智能体适应方案和复习计划。

## 二、测试环境

- 测试系统：Windows PowerShell
- 运行方式：Python CLI
- 模型模式：豆包 API
- API Key 配置：通过 `ARK_API_KEY` 环境变量配置
- 默认课程资料：`data/sample_course_notes.md`
- 空课程资料：`data/empty_notes.md`
- 输出报告：`outputs/summary_report.md`

## 三、测试用例与测试记录

| 测试编号 | 测试场景 | 输入 | 预期结果 | 实际结果 | 是否通过 | 改进建议 |

## 四、测试结论

当前版本已取消本地 mock 问答模式，主流程只调用豆包 API。测试结果表明，系统可以根据不同学习问题生成不同的问题类型、资料摘要、核心知识点、回答内容、适应方案和复习计划，较好体现了智能体工作流的动态适应能力。

项目仍可继续优化，例如增加多轮追问、支持更多资料格式、增加资料依据充分性评分、完善飞书签名校验和自动化测试脚本。
